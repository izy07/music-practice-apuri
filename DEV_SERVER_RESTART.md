# 開発サーバー再起動手順

## 🔍 問題

`ERR_NETWORK_IO_SUSPENDED` エラーが発生する場合、パソコンがスリープ状態になったことで開発サーバーの接続が中断されています。

## ✅ 解決方法

### 方法1: 簡単な再起動（推奨）

```bash
# 1. 開発サーバーを起動（キャッシュクリア付き）
npm run web:clean

# または通常の起動
npm run web
```

### 方法2: 手動でポートを解放して再起動

```bash
# 1. ポート8081を使用しているプロセスを確認
lsof -ti:8081

# 2. プロセスを終了（出力されたPIDを使用）
kill <PID>

# 3. 開発サーバーを再起動
npm run web:clean
```

### 方法3: 異なるポートで起動

```bash
# ポート8082で起動
npm run web:port

# ブラウザで http://localhost:8082 にアクセス
```

## 🚀 推奨ワークフロー

1. **開発サーバーを起動**:
   ```bash
   npm run web
   ```

2. **問題が発生した場合**:
   ```bash
   # Ctrl+C でサーバーを停止
   # その後、キャッシュクリアして再起動
   npm run web:clean
   ```

3. **スリープ復帰後の対処**:
   - パソコンがスリープから復帰したら、開発サーバーを再起動することを推奨
   - または、スリープを無効化する設定を検討

## ⚠️ 注意事項

- スリープ復帰後は接続が不安定になることがあります
- 長時間開発する場合は、スリープ設定を調整するか、開発サーバーを定期的に再起動してください
