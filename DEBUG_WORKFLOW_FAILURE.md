# ワークフロー失敗のデバッグ

## 🔴 現在の状況

**Deploy to GitHub Pages #11**: 失敗
- **build ジョブ**: 失敗（exit code 1）
- **deploy ジョブ**: スキップ
- **実行時間**: 19秒

## 🔍 エラーの詳細を確認

### ステップ1: build ジョブのログを確認

1. **Actions タブで #11 をクリック**
2. **"build" ジョブをクリック**
3. **各ステップのログを確認**

特に確認すべきステップ：
- ✅ "依存関係のインストール" - `npm ci --legacy-peer-deps` が成功したか
- ❌ "Webビルド（GitHub Pages用）" - ビルドエラーが発生していないか
- ❌ その他のステップ

### ステップ2: エラーメッセージを特定

ログから以下を確認：
- どのステップで失敗したか
- エラーメッセージの内容
- スタックトレース（あれば）

## 🔧 よくある原因

### 1. ビルドスクリプトのエラー

**エラーメッセージ例:**
```
Error: Command failed: npm run build:web:github
```

**対処法:**
- ローカルで `npm run build:web:github` を実行して確認
- スクリプトファイルの存在を確認

### 2. 環境変数の問題

**エラーメッセージ例:**
```
Error: EXPO_PUBLIC_WEB_BASE is not defined
```

**対処法:**
- ワークフローファイルで環境変数が正しく設定されているか確認

### 3. スクリプトファイルのパスエラー

**エラーメッセージ例:**
```
Error: Cannot find module 'scripts/fix-github-pages-paths.js'
```

**対処法:**
- スクリプトファイルが存在するか確認
- ファイルパスが正しいか確認

## 📋 次のステップ

1. **build ジョブのログを確認**
2. **エラーメッセージをコピー**
3. **エラーの内容を共有**

これで、具体的な修正方法を提案できます。
